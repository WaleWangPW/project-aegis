# P1D.1 — Stock Agent Recommendation Explanation Guide

## Purpose

This guide tells the Stock Agent (Feishu / OpenClaw) how to answer
questions about Project Aegis recommendation details without giving trading
advice and without accessing raw JSONL records.

---

## File to read

When the user asks about recommendation details, read:

```
/Users/weihongwang/.openclaw/agents/stock-agent/workspace/project-aegis/recommendation_details.json
```

This file is a sanitized, read-only mirror generated by
`aegis/desktop/recommendation_details.py`. It is refreshed every time
`scripts/refresh_stock_agent_aegis_status.py` runs.

**Do NOT read `data/records/*.jsonl` directly.** Those are the raw repo
records; the stock-agent workspace does not have access to them, and the
mirror exists precisely to avoid that.

---

## How to answer

When the user asks something like:

> 请说明当前 Project Aegis 是否生成了 Recommendation。
> 如果有，请列出每条 Recommendation 的 status、symbol、support reasons、
> oppose reasons、risks、invalidation conditions 和 why_not_action。
> 不要给交易建议，只解释系统记录。

1. Read `recommendation_details.json`.
2. Check `summary.total_recommendations`. If 0, say no recommendations
   have been generated yet.
3. For each entry in `recommendations` where `is_latest_for_symbol` is
   `true`, report the following fields:

   - **symbol** — the ticker (e.g. CRCL)
   - **name** — company name
   - **market** — US / A / H
   - **date** and **session** — when the recommendation was generated
   - **status** — Action / Ready / Watch / Exit
   - **confidence** — a number 0.0–1.0
   - **decision_summary** — the engine's own one-line summary
   - **support_reasons** — list (may be empty)
   - **oppose_reasons** — list (may be empty)
   - **risks** — list (may be empty)
   - **invalidation_conditions** — list (may be empty)
   - **why_not_action** — string or null; explains why status is not Action
   - **expert_opinions** — per-expert stance, confidence, summary,
     missing_data
   - **data_quality_notes** — data gaps relevant to this symbol

4. If `is_latest_for_symbol` is `false` for some entries, those are older
   runs of the same recommendation cycle; do not surface them unless the
   user explicitly asks for history.

---

## Language rules

- Say **"Project Aegis 系统记录显示 status=Exit"** — never "你应该退出".
- Say **"系统判断 why_not_action=risk_veto_triggered"** — never "建议你…".
- If confidence is low (< 0.5), state it explicitly:
  "当前 confidence 为 0.25，系统判断依据不足。"
- If data gaps drove the decision, state that clearly:
  "多个 Expert Agent 因缺少数据（trend_signal、risk_signal 等）返回 neutral，
  RiskAgent 因 liquidity_not_ok 发出 veto，导致系统输出 Exit。"
- Never say "buy" or "sell" as an instruction.
- Never give a price target or position size.
- Never say "I recommend" or "you should".

---

## Handling missing data honestly

If `support_reasons` is empty, say so: "系统记录中 support_reasons 为空。"

If `oppose_reasons` is empty, say so.

If `why_not_action` is null, say: "系统未记录 why_not_action。"

Do not invent reasons. Do not say "probably because of X" unless X appears
in the JSON.

---

## CRCL is not special-cased

CRCL is treated identically to any other symbol. Do not apply different
logic, different thresholds, or different language for CRCL entries.
